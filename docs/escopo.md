# Escopo do Projeto

## 1. Visão Geral

Este projeto consiste no desenvolvimento de um **Sistema de Controle de Medicamentos Doméstico**, voltado ao auxílio de usuários no gerenciamento do uso diário de medicamentos. O sistema tem como foco reduzir esquecimentos, atrasos e usos inadequados, oferecendo um controle simples, organizado e acessível, especialmente para uso doméstico.

O sistema não possui finalidade clínica, diagnóstica ou hospitalar, atuando exclusivamente como uma ferramenta de apoio à organização pessoal do usuário.

---

## 2. Público-Alvo

O sistema é destinado a usuários domésticos, com ênfase em:

* Idosos que fazem uso contínuo de medicamentos;
* Pessoas que utilizam múltiplos medicamentos diariamente;
* Usuários com dificuldade em gerenciar horários e recomendações associadas ao uso de medicamentos.

---

## 3. Funcionalidades Abrangidas (Dentro do Escopo)

O sistema deverá contemplar as seguintes funcionalidades:

* Cadastro de medicamentos, contendo informações básicas como nome, dosagem e frequência de uso;
* Definição de horários para administração dos medicamentos;
* Registro de recomendações associadas ao uso (antes ou após refeições, intervalos específicos, observações);
* Emissão de alertas ou lembretes nos horários programados;
* Registro da confirmação de uso pelo usuário;
* Visualização de um histórico simples de medicamentos administrados.

---

## 4. Funcionalidades Não Abrangidas (Fora do Escopo)

As seguintes funcionalidades **não fazem parte** do escopo deste projeto:

* Prescrição médica ou validação clínica de medicamentos;
* Diagnóstico, acompanhamento médico ou análise de efeitos colaterais;
* Integração com sistemas hospitalares, clínicas ou farmácias;
* Compra ou renovação de medicamentos;
* Armazenamento em nuvem ou sincronização entre múltiplos dispositivos;
* Compartilhamento de dados com terceiros.

---

## 5. Limitações do Sistema

* O sistema será desenvolvido para uso individual e doméstico;
* As informações inseridas são de responsabilidade exclusiva do usuário;
* O sistema não substitui orientações médicas ou farmacêuticas;
* O funcionamento prioriza o armazenamento local de dados.

---

## 6. Considerações Finais

Este escopo estabelece limites claros para o desenvolvimento do sistema, garantindo um projeto viável, controlado e alinhado ao objetivo acadêmico proposto. Funcionalidades não previstas poderão ser consideradas em evoluções futuras, fora do contexto deste projeto.
