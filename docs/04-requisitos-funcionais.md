# Requisitos Funcionais do Sistema

### Logo abaixo é possível observar as funções e ações do sistema:

* RF1 – Cadastro de medicamentos
O sistema deve permitir o cadastro de medicamentos, possibilitando informar nome, tipo de dosagem (ex.: comprimido, cápsula, líquido), quantidade por administração e duração do tratamento.

* RF2 – Definição de horários e intervalos de uso
O sistema deve permitir a definição de horários de administração, bem como o cálculo automático de intervalos de uso com base na frequência informada pelo usuário.

* RF3 – Definição de período de tratamento
O sistema deve permitir ao usuário definir a data de início e a data de término do uso do medicamento, incluindo tratamentos com duração determinada (ex.: 7 dias, 14 dias).

* RF4 – Registro de recomendações e observações
O sistema deve permitir o registro de observações fornecidas pelo usuário ou sugeridas pelo sistema, tais como uso antes ou após refeições, modo de administração, precauções e restrições.

* RF5 – Notificação de horários programados
O sistema deve notificar o usuário nos horários programados para a administração dos medicamentos cadastrados.

* RF6 – Confirmação de uso do medicamento
O sistema deve permitir que o usuário registre a confirmação da administração do medicamento no horário indicado.

* RF7 – Visualização do histórico de uso
O sistema deve exibir de forma clara o histórico de administrações realizadas e não realizadas.

* RF8 – Acompanhamento do progresso do tratamento
O sistema deve permitir a visualização do progresso do uso do medicamento, indicando doses já administradas, doses pendentes e a proximidade do término do tratamento.

* RF9 – Gerenciamento de múltiplos medicamentos
O sistema deve permitir o gerenciamento simultâneo de múltiplos medicamentos, respeitando os horários e intervalos definidos individualmente.
