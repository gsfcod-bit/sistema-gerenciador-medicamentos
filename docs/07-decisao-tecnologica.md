# Decisão Tecnológica

## 1. Visão Geral da Solução Tecnológica

Este projeto propõe o desenvolvimento de um sistema de controle de medicamentos doméstico baseado em tecnologias web, executado inteiramente no lado do cliente (client-side), sem dependência de servidores ou serviços externos. A solução prioriza simplicidade, acessibilidade e viabilidade dentro do escopo definido.

---

## 2. Premissas Técnicas do Projeto

As decisões tecnológicas adotadas consideram as seguintes premissas:

- Execução local no navegador do usuário;
- Ausência de backend ou processamento em servidor;
- Persistência local de dados;
- Compatibilidade com diferentes tamanhos de tela;
- Facilidade de manutenção e compreensão do código;
- Adequação ao contexto acadêmico e ao público-alvo do sistema.

---

## 3. Tecnologias Selecionadas

### 3.1 HTML

O HTML será utilizado para estruturar semanticamente as informações do sistema, garantindo organização lógica do conteúdo, acessibilidade e base sólida para manipulação dinâmica via JavaScript.

---

### 3.2 CSS

O CSS será responsável pela apresentação visual da aplicação, assegurando clareza, legibilidade e adaptação a diferentes dispositivos, por meio de layouts responsivos e organização visual consistente.

---

### 3.3 JavaScript

O JavaScript será utilizado como a principal tecnologia para implementação da lógica do sistema, incluindo:

- Manipulação dinâmica da interface;
- Gerenciamento dos dados em memória;
- Controle de eventos e interações do usuário;
- Implementação das regras de negócio;
- Persistência de dados localmente.

---

## 4. Persistência de Dados

A persistência das informações será realizada localmente no ambiente do navegador, permitindo que os dados permaneçam disponíveis entre sessões de uso, sem necessidade de conexão com servidores externos.

Essa abordagem atende às necessidades do sistema e mantém a simplicidade arquitetural proposta.

---

## 5. Arquitetura da Aplicação

A aplicação seguirá uma arquitetura simples e modular, com separação de responsabilidades entre:

- Estrutura (HTML);
- Apresentação (CSS);
- Lógica e comportamento (JavaScript).

Essa separação visa facilitar a manutenção, leitura e evolução do sistema.

---

## 6. Justificativa da Escolha Tecnológica

A adoção de HTML, CSS e JavaScript puro se justifica por:

- Compatibilidade com os requisitos funcionais e não funcionais definidos;
- Baixa complexidade arquitetural;
- Facilidade de aprendizado e domínio técnico;
- Independência de frameworks e bibliotecas externas;
- Adequação ao escopo e aos objetivos do projeto.

---

## 7. Limitações Tecnológicas

As decisões tecnológicas adotadas implicam nas seguintes limitações:

- Execução restrita ao ambiente do navegador;
- Ausência de sincronização entre dispositivos;
- Dependência do dispositivo local para armazenamento dos dados;
- Funcionalidades limitadas às capacidades do ambiente web client-side.

---

## 8. Considerações Finais

A decisão tecnológica adotada visa equilibrar viabilidade, aprendizado e funcionalidade, garantindo que o sistema atenda às necessidades propostas de forma clara, organizada e tecnicamente consistente, mantendo-se dentro dos limites estabelecidos pelo escopo do projeto.
