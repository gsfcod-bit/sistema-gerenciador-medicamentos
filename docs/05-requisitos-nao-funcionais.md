Requisitos Não Funcionais do Sistema

Os requisitos não funcionais definem os atributos de qualidade, restrições e características comportamentais do sistema, garantindo sua usabilidade, confiabilidade e adequação ao público-alvo.

* RNF1 – Usabilidade e simplicidade da interface
O sistema deve possuir uma interface simples, intuitiva e de fácil compreensão, priorizando clareza visual e facilidade de uso, especialmente para usuários com pouca familiaridade com tecnologia.

* RNF2 – Responsividade e adaptação a diferentes telas
O sistema deve adaptar sua interface para diferentes tamanhos e resoluções de tela, garantindo usabilidade adequada em dispositivos móveis, tablets e computadores.

* RNF3 – Desempenho e leveza da aplicação
O sistema deve ser leve, com recursos visuais otimizados, de modo a não comprometer o desempenho mesmo em dispositivos com hardware limitado.

* RNF4 – Confiabilidade e tolerância a falhas
O sistema deve apresentar comportamento estável e previsível, sendo resistente a falhas de execução e garantindo a integridade dos dados armazenados.

* RNF5 – Efetividade das notificações
O sistema deve garantir que os alertas e lembretes sejam apresentados de forma clara e perceptível, utilizando diferentes formas de aviso quando necessário, de modo a reduzir o risco de esquecimento pelo usuário.
